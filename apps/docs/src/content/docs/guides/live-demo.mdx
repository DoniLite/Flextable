## What's the same, what's different

The `ColumnFactory` calls building these two tables are close to identical — same methods, same
options, same mock data. The only per-framework differences are:

- `toReactColumnDef()` vs. `toVueColumnDef()` to turn `ColumnConfig`s into real `ColumnDef`s.
- `@flextable/react-ui` vs. `@flextable/vue-ui` supplying the default cell components.
- Reading the full source for each is the fastest way to see exactly how little changes between
  the two bindings — see
  [`ReactTableDemo.tsx`](https://github.com/DoniLite/Flextable/blob/main/apps/docs/src/components/demo/ReactTableDemo.tsx)
  and
  [`VueTableDemo.vue`](https://github.com/DoniLite/Flextable/blob/main/apps/docs/src/components/demo/VueTableDemo.vue).
